- Compute Engine

IassS offering

Predefined and customized VMs

Persistent disks and local SSDs

Managed instance groups -> autoscaling can be enables

Per-second billing

Popular choice because:

- complete control over infrastructure
- on premise workloads can easily be lifted and shifted


- GKE

Containerized applications
A container is code packaged with its dependecies


- App Engine

Fully managed PaaS offering

Focused on application logic

Developers just upload code and it takes care of the required infrastructure

Good choice for developers that: 
- Want to focus on writing code
- focus on building applications rather than deploying and managing environment

- Cloud Function

Event based, asynchronous compute solution
Executes code in response to events

Deploys the computing capacity to run code

Can be part of microservices application


- Cloud run

Managed compute platform 

Runs stateless containers

Serverless

It's fast
